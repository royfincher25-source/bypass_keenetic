## 🔍 Проблема

**Было обнаружено:**
1. **BusyBox xargs не поддерживает `-P`** — все "параллельные" запросы выполнялись последовательно
2. **`dig` медленный** на Entware — 1.1 сек на запрос
3. **Избыточный код** — кэширование, проверки, сложные циклы

---

## ✅ Решение

### 1. Параллелизм через `&` (фон)

```bash
# Было (НЕ работает):
xargs -P20 -I{} dig +short {} @8.8.8.8

# Стало (работает):
dig +short "$domain" @8.8.8.8 &
pids="$pids $!"
# ... ждём каждые 20 процессов
for pid in $pids; do wait $pid; done
```

### 2. nslookup вместо dig

```bash
# Было (медленно):
dig +short google.com @8.8.8.8 +time=2  # 1.1 сек

# Стало (быстро):
nslookup google.com 8.8.8.8  # 0.07 сек
```

### 3. Упрощение кода

- Убрано кэширование (домены часто меняются)
- Убраны лишние проверки
- Пакетная загрузка в ipset через `restore -!`

## 🔧 Решение проблем

### Скрипт зависает

**Причина:** Слишком высокая параллельность

```bash
# Уменьшить MAX_PARALLEL
MAX_PARALLEL=10  # было 20
```

### Мало IP в ipset

**Причина:** DNS не возвращает IP

```bash
# Проверка nslookup
nslookup google.com 8.8.8.8 | grep -i 'address'

# Если пусто — проверить DNS
ping 8.8.8.8
```

### Долгая первая загрузка

**Нормально:** 200 доменов × 1 сек = ~200 сек (последовательно)
**С параллельностью 20:** ~10-15 сек
