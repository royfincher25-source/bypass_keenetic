# 📋 План: Web-интерфейс для bypass_keenetic_web

**Дата:** 11 марта 2026 г.  
**Статус:** План  
**Версия:** 1.0  
**Проект:** `bypass_keenetic_web`

---

## Цель

Заменить Telegram-бота на web-интерфейс с тем же функционалом.

**Причина:** Если Telegram заблокируют, останется только web-интерфейс.

---

## Требования

1. **Web — основной интерфейс**, Telegram удалить
2. **Локальный доступ** (192.168.1.1:8080)
3. **Функционал** — меню как в Telegram (1:1)

---

## Архитектура

### Стек

- **Flask 3.0** — web-фреймворк (лёгкий, простой)
- **Jinja2** — шаблоны
- **Bootstrap 5.3** — адаптивный UI, тёмная тема
- **Session cookie** — авторизация

### Структура

```
bypass_keenetic_web/
├── src/
│   ├── core/           # Общий: конфиги, логика, утилиты
│   └── web/            # Новый web-интерфейс
│       ├── app.py      # Точка входа
│       ├── routes.py   # Маршруты
│       ├── templates/  # HTML шаблоны
│       │   ├── base.html
│       │   ├── index.html
│       │   ├── keys.html
│       │   ├── bypass.html
│       │   ├── install.html
│       │   ├── stats.html
│       │   └── service.html
│       └── static/     # CSS, JS
│           └── style.css
├── requirements.txt
└── README.md
```

### Технические решения

| Решение | Выбор |
|---------|-------|
| **Навигация** | Плитки (cards) |
| **Тема** | Тёмная (Bootstrap dark) |
| **Динамика** | Полные перезагрузки страниц |
| **Адаптивность** | Bootstrap grid (авто) |
| **Авторизация** | Session cookie + пароль |
| **Порт** | 8080 (локально) |

## Этапы реализации

### 1. Подготовка проекта

- Создать структуру `bypass_keenetic_web/`
- Инициализировать git
- Создать `requirements.txt`
- Настроить `.env.example`

### 2. Создание core-модуля

- Вынести общую логику из `bot3/` в `core/`
- Создать `core/config.py` (парсер .env)
- Создать `core/handlers.py` (бизнес-логика)

### 3. Создание web-приложения

- Flask приложение (app.py)
- Маршруты (routes.py)
- Session middleware
- Авторизация (login/logout)

### 4. UI компоненты (шаблоны)

- `base.html` — базовый шаблон (тёмная тема)
- `index.html` — главное меню (плитки)
- `keys.html` — ключи и мосты
- `bypass.html` — списки обхода
- `install.html` — установка/удаление
- `stats.html` — статистика
- `service.html` — сервисное меню
- `login.html` — вход

### 5. Интеграция

- Запуск на порту 8080
- Логирование
- Обработка ошибок

### 6. Тестирование

- Проверка всех маршрутов
- Проверка авторизации
- Проверка на роутере

### 7. Документация

- README.md
- Инструкция по установке
