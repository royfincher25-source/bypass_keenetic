# 📦 Архивирование и восстановление бэкапа

**Версия:** 3.5.6  
**Дата:** 9 марта 2026 г.

---

## 🎯 Назначение

Создание резервных копий конфигурации бота для:
- Восстановления после сбоя
- Переноса на другой роутер
- Сохранения настроек перед обновлением

---

## 📋 Что входит в бэкап

### ✅ Сохраняется:

| Файл/Папка | Назначение |
|------------|------------|
| `/opt/etc/bot/` | Все файлы бота (main.py, handlers.py, utils.py, bot_config.py) |
| `/opt/etc/bot/core/` | Core модули (config.py, env_parser.py, http_client.py, etc.) |
| `/opt/etc/bot/.env` | Конфигурация (токен, настройки) |
| `/opt/etc/unblock/*.txt` | Списки обхода |
| `/opt/etc/tor/torrc` | Конфигурация Tor |
| `/opt/etc/shadowsocks.json` | Конфигурация Shadowsocks |
| `/opt/etc/xray/config.json` | Конфигурация Xray/VLESS |
| `/opt/etc/trojan/config.json` | Конфигурация Trojan |
| `/opt/root/script.sh` | Скрипт установки |
| `/opt/root/KeenSnap/keensnap.sh` | Скрипт бэкапирования |
| `/opt/etc/crontab` | Расписание задач |
| `/opt/etc/init.d/S99telegram_bot` | Init скрипт |
| `/opt/etc/init.d/S99unblock` | Init скрипт разблокировки |

### ❌ Не сохраняется:

| Файл/Папка | Причина |
|------------|---------|
| `__pycache__/` | Кэш Python (генерируется заново) |
| `error.log` | Логи (не нужны для восстановления) |
| `/opt/root/bot_backup_*/` | Старые бэкапы |

### Способ 2: Через SSH (для продвинутых)

#### Шаг 1: Подключиться к роутеру

```bash
ssh admin@192.168.1.1
```

---

#### Шаг 2: Найти архив

```bash
ls -la /mnt/disk1/backup_*.tar.gz
```

**Пример вывода:**
```
-rw-r--r-- 1 root root 47185920 Mar  9 17:30 /mnt/disk1/backup_2026-03-09_17-30.tar.gz
```

---

#### Шаг 3: Распаковать архив

```bash
# Перейти в корень
cd /

# Распаковать архив
tar -xzf /mnt/disk1/backup_2026-03-09_17-30.tar.gz
```

---

#### Шаг 4: Установить права

```bash
# Скрипты
chmod 755 /opt/root/script.sh
chmod 755 /opt/root/KeenSnap/keensnap.sh
chmod 755 /opt/etc/init.d/S99telegram_bot
chmod 755 /opt/etc/init.d/S99unblock

# Python файлы
chmod 644 /opt/etc/bot/*.py
chmod 644 /opt/etc/bot/core/*.py

# Конфигурация
chmod 600 /opt/etc/bot/.env
```

---

#### Шаг 5: Перезапустить бота

```bash
/opt/etc/init.d/S99telegram_bot restart
```

---

#### Шаг 6: Проверить логи

```bash
tail -50 /opt/etc/bot/error.log
```

**Ожидаемый результат:**
```
2026-03-09 18:00:00 - Бот запущен (PID: 12345)
2026-03-09 18:00:01 - Загрузка конфигурации...
2026-03-09 18:00:02 - Бот готов к работе
```

## 💡 Советы

### 📅 Регулярность бэкапирования

| Событие | Бэкап |
|---------|-------|
| Перед обновлением | ✅ Обязательно |
| После изменения настроек | ✅ Рекомендуется |
| Раз в месяц | ✅ Для безопасности |
| Перед перепрошивкой | ✅ Обязательно |

---

### 💾 Хранение бэкапов

**Рекомендации:**
- Хранить **2-3 последних** бэкапа
- Удалять старые (> 1 месяца)
- Копировать на **внешний носитель** (облако, ПК)

---

### 🔒 Безопасность

**Важно:**
- `.env` содержит **токен бота** — не передавайте никому
- Архивы храните в **защищённом месте**
- При переносе используйте **шифрование**

## ❓ Часто задаваемые вопросы

### Q: Где хранятся бэкапы?

**A:** На выбранном диске в корне:
```
/mnt/disk1/backup_2026-03-09_17-30.tar.gz
```

---

### Q: Сколько места занимает бэкап?

**A:** ~30-70 MB (в сжатом виде)

---

### Q: Можно ли восстановить бэкап на другом роутере?

**A:** Да, но нужно:
1. Установить ту же версию бота
2. Обновить `.env` (токен, настройки)
3. Проверить пути к файлам

---

### Q: Что делать, если бэкап не создаётся?

**A:**
1. Проверить место на диске:
   ```bash
   df -h /mnt/disk1
   ```
2. Проверить логи:
   ```bash
   tail -50 /opt/etc/bot/error.log
   ```
3. Перезапустить бота:
   ```
   /restart
   ```

---

### Q: Как удалить старый бэкап?

**A:** Через SSH:
```bash
rm /mnt/disk1/backup_2026-03-01_12-00.tar.gz
```
