## 1. Проблемы старого логирования

### 1.1 Блокирующая запись

**Старый код:**
```python
def log_error(message):
    with open(log_file, "a") as f:
        f.write(f"{timestamp} - {message}\n")
```

**Проблемы:**
- ❌ Блокировка потока на время записи
- ❌ Задержка при каждом вызове
- ❌ Риск потери данных при сбое

---

### 1.2 Отсутствие ротации

**Проблемы:**
- ❌ Лог растёт бесконечно
- ❌ Заполнение диска
- ❌ Сложность анализа больших файлов

---

### 1.3 Простой текстовый формат

**Проблемы:**
- ❌ Сложность парсинга
- ❌ Нет структурированных данных
- ❌ Нет контекста (модуль, функция, PID)

---

## 2. Новое асинхронное логирование

### 2.1 Архитектура

```
┌─────────────────────────────────────────┐
│  Приложение (поток 1)                   │
│  logger.error("Ошибка")  ← Неблокирующе │
└────────────────┬────────────────────────┘
                 │
                 ▼
         ┌───────────────┐
         │  Queue (100)  │  ← Очередь логов
         └───────┬───────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  Worker Thread (поток 2)                │
│  - Получает из очереди                  │
│  - Пишет в файл                         │
│  - Ротация при необходимости            │
└─────────────────────────────────────────┘
```

---

### 2.2 Преимущества

| Преимущество | Описание |
|--------------|----------|
| **Неблокирующая запись** | Вызов `log_error()` не блокирует поток |
| **Автоматическая ротация** | Лог не превышает max_size |
| **JSON формат** | Структурированное логирование |
| **Уровни логирования** | DEBUG, INFO, WARNING, ERROR, CRITICAL |
| **Потокобезопасность** | Корректная работа в многопоточной среде |
| **Минимальные накладные расходы** | ~0.01 мс на вызов vs ~5 мс у синхронного |

## 3. Использование

## 4. Ротация логов

### 4.1 Автоматическая ротация

**Когда срабатывает:**
- Размер файла превышает `max_size` (по умолчанию 512 KB)

**Процесс:**
```
error.log      → error.log.1  (свежий бэкап)
error.log.1    → error.log.2
error.log.2    → error.log.3  (старый бэкап)
error.log.3    → Удалён
```

**Новый файл:** `error.log` (пустой)

---

### 4.2 Настройка ротации

```python
logger = init_logger(
    log_file='/opt/etc/bot/error.log',
    max_size=1024 * 1024,   # 1 MB
    max_backups=5           # 5 бэкапов
)
```

**Максимальный размер всех логов:**
```
max_size × max_backups = 1 MB × 5 = 5 MB
```

## 8. Диагностика проблем

### 8.1 Логгер не пишет

**Проверка:**
```python
from core.logging_async import get_logger

logger = get_logger()
if logger:
    logger.info("Тест")
    logger.flush()  # Принудительная запись
else:
    print("Логгер не инициализирован!")
```

---

### 8.2 Очередь переполнена

**Симптом:**
- Пропускаются записи в логе

**Решение:**
```python
# Увеличить размер очереди
logger = init_logger(queue_size=500)
```

---

### 8.3 Ротация не срабатывает

**Проверка:**
```python
import os
log_file = '/opt/etc/bot/error.log'
print(f"Размер: {os.path.getsize(log_file)} байт")
print(f"Бэкапы: {[f for f in os.listdir('/opt/etc/bot') if 'error.log' in f]}")
```
